## 2024-03-01 - Avoid synchronous JSON.parse on large spatial data in React render cycles
**Learning:** Parsing large JSON strings containing polyline arrays (`JSON.parse(polylineString)`) inside a function called repeatedly or within a UI element event handler block the main thread and degrade performance, particularly when the data is large spatial coordinates. `downloadGPXFromPolyline` was taking a stringified polyline, but React components (`RoutePanel`, `MapCanvas`) had to `JSON.stringify` the existing array from `routeData.polyline` just to pass it to `downloadGPXFromPolyline`, which immediately called `JSON.parse` again.
**Action:** Always maintain large structural data (like coordinates and polylines) as objects/arrays rather than strings wherever possible to avoid redundant serialize/deserialize roundtrips. Refactored `downloadGPXFromPolyline` to accept an array directly, eliminating unnecessary stringify-parse operations and avoiding blocking the main thread during GPX generation.
