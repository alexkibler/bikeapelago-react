## 2024-06-25 - Expensive JSON Parsing in Render Loop
**Learning:** Found synchronous `JSON.parse` operations running on potentially large spatial data (polylines) directly inside the React component render loop in `GameView.tsx`. This causes the main thread to block and results in laggy UI interactions whenever the component re-renders (e.g., state updates not related to the polyline).
**Action:** Always extract and memoize `JSON.parse` and array `.map()` transformations of large datasets using `useMemo` so they only recalculate when the source data actually changes.
## 2024-04-15 - Unnecessary JSON stringify/parse on large array
**Learning:** Found a component (`RoutePanel.tsx`) that used `JSON.stringify` to pass an array of coordinates to a helper function (`downloadGPXFromPolyline`), which immediately called `JSON.parse` to convert it back to an array. For spatial datasets like routing polylines, which can contain thousands of points, this is a significant bottleneck causing a main-thread CPU spike. Furthermore, it led to a runtime bug where `MapCanvas.tsx` passed the raw array directly, failing the `JSON.parse` at runtime or failing type-checks.
**Action:** Always verify if a function actually needs a serialized string, or if it can accept the data structure directly. Avoid arbitrary serialization of objects/arrays across internal boundaries unless necessary for storage or network transport.
